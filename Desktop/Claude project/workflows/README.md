# Workflows

Markdown SOPs that define what to do and how. Each workflow should include:

- **Objective** – what this workflow accomplishes
- **Required Inputs** – what's needed before starting
- **Steps** – ordered list of actions, referencing tools where relevant
- **Expected Outputs** – what success looks like
- **Edge Cases** – known failure modes and how to handle them

Workflows are living documents. Update them as you learn.
